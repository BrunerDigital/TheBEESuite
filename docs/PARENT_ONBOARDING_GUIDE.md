# The Bee Suite Parent Onboarding Guide

Use this guide when a school is ready to invite families into the parent portal.

## Director Steps

1. Confirm the family profile exists in `Family Profiles`.
2. Confirm each child is linked to the correct family and classroom.
3. Confirm the primary guardian has a valid email address.
4. Set or confirm the guardian's 4 digit kiosk PIN if lobby check-in is active.
5. In `Family Profiles`, open `Parent Portal Access`.
6. Select `Invite Parent` for the guardian.
7. Leave the temporary password field blank to send a password setup/reset email, or enter a temporary password if the school wants to hand it to the parent directly.
8. Ask the parent to sign in at `/parent-portal`.

## What Parents Can See

- Family dashboard
- Linked children
- Classroom and schedule snapshot
- Daily reports with meals, naps, potty/diaper logs, activities, notes, and supplies
- Photos shared by teachers after school review
- Open invoices, balances, recent payments, and ledger history
- Tuition checkout buttons for ACH or card when Stripe is configured for the school
- Incident reports requiring acknowledgement
- Documents requiring submission or review
- Messages and announcements
- Notification preferences

## Safety And Privacy Rules

- A parent account only sees a family when its user account is linked to a guardian record.
- Sensitive documents stay restricted and are submitted for director review.
- Parents can request emergency contact or pickup changes, but the school must review the request before operational records should be changed.
- AI assistance must not make final safety, medical, custody, billing, or compliance decisions.

## Troubleshooting

- If a parent cannot log in, resend portal access from `Family Profiles`.
- If the parent logs in but sees no family, confirm the guardian record has the correct email and is linked to the parent user.
- If payment buttons show an error, confirm the school has completed Stripe payout onboarding and parent payments are enabled.
- If photos are missing, confirm the child has photo/video permission and the media item is approved/shared.
