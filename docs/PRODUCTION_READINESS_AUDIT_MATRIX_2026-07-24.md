## Evidence snapshot

- Repository: `main` at `83b6d74f7183be09cad8e327d7308853a0dbeea7`, equal to `origin/main` at discovery.
- User-owned dirty work preserved: `src/app/[slug]/page.tsx`, `src/components/billing-workbench.tsx`, `src/components/enrollment-visibility-panels.tsx`, `src/components/family-record-editor.tsx`, and three untracked ProCare data directories.
- Stack: Next.js 16 App Router, React 19, Prisma/Postgres, Supabase Auth/Storage, Stripe, SendGrid, Twilio, Vercel, Capacitor iOS.
- Inventory: 25 page files, 125 API route files under `src/app/api`, 86 Prisma models, 33 Prisma migration directories, 8 configured cron routes, and 162 test files.
- Static API review: all but eight API route files contain an obvious authenticated, cron, public, or signed-webhook boundary. The eight exceptions are the expected auth, health, kiosk, and public survey response surfaces and require their own rate-limit/token rules rather than ordinary session auth.
- Baseline checks on July 24: lint passed; typecheck passed; operations check passed with 8/8 cron parity and 33/33 migration file presence; the final full test suite passed 584/584.
- Full production gate passed after the framework, survey, billing-family, ProCare, and inquiry-origin fixes: Prisma Client generation, lint, typecheck, 584/584 tests, and Next.js 16.2.11 production build with all 153 static pages generated.
- Supply-chain remediation: Next.js `16.2.6` was vulnerable to the July 2026 security set. The local candidate now pins Next.js and `eslint-config-next` to `16.2.11` and overrides Next's optional Sharp dependency to `0.35.3`. `npm audit --omit=dev` reports zero vulnerabilities. Compatible transitive development-tool patches reduced the all-dependency audit to one low-severity local Windows development-server advisory in `tsx@4.21.0 -> esbuild@0.27.7`; no compatible upstream `tsx` release currently removes it. This candidate is not deployed.
- Data-model finding: `Family.centerId` is nullable and has no Prisma relation/foreign-key constraint to `Center`. Existing application guards often scope through the scalar value, but database-level orphan prevention is absent. Do not add a production constraint until a read-only orphan/null inventory and backward-compatible remediation plan are approved.
- Read-only production count audit found zero families without centers, zero orphan family-center IDs, zero guardian-user tenant mismatches, zero staff-user tenant mismatches, zero document family/child mismatches, zero survey family/center or tenant mismatches, zero duplicate family source keys, and zero duplicate invoice numbers. It found 49 child/classroom and 12 staff/classroom center mismatches, all inside `bee-suite-demo` and not cross-tenant; two user/grant tenant mismatches and one grant/center tenant mismatch, all on inactive grants; 31 duplicated ProCare guardian source-key groups affecting 67 records; and 150 duplicated ProCare child source-key groups affecting 555 records. No data was changed.
- No production database write, migration, email, SMS, push, invoice, charge, refund, payout, account change, invitation, or deployment was performed.
