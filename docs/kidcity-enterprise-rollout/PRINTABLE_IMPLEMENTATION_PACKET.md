## One-Page Rollout Summary

This school is joining The Bee Suite enterprise rollout. The school will run ProCare and The Bee Suite at the same time while records, attendance, billing, parent access, staff workflows, devices, and reporting are validated.

The school may stop using ProCare only after:

- Director signoff.
- Corporate checklist completion.
- Data verification completion.
- Attendance and kiosk verification.
- Billing verification.
- Stripe/payment approval if payments are included.
- Staff readiness.
- Parent adoption threshold.
- No unresolved P0/P1 issues.
- No critical bugs for 14 consecutive days.
- Executive leadership approval.

## Support Escalation Card

Report immediately:

- Wrong school, family, child, classroom, invoice, document, medical, custody, or payment data.
- Kiosk check-in/check-out failure.
- Missing active children.
- Incorrect billing balances.
- Payment or payout issue.
- Login outage.

Include:

- School.
- User email and role.
- Page or workflow.
- Time and timezone.
- Expected result.
- Actual result.
- Screenshot only if safe.

## Rollback Summary

If a school must temporarily return to ProCare for a workflow:

1. Declare rollback scope.
2. Pause affected Bee Suite workflow.
3. Tell staff which system is official.
4. Preserve Bee Suite records created since the rollback point.
5. Reconcile official records into ProCare if ProCare is the control system.
6. Fix and re-test.
7. Resume only after approval.
