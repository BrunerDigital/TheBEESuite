# Executive Admin Quick Start

Last updated: July 24, 2026

This guide is for Kid City USA corporate/executive users and future brand/franchise administrators.

## Daily Overview

- Use the executive dashboard for cross-location CRM, FTE, reporting, and operational snapshots.
- Use filters by location, state, region, owner group, and date where available.
- Treat demo classroom/parent engagement data as executive-only preview data until each school's real operational data is imported and validated.

## Location And User Management

- Add or archive locations from the executive/admin console when schools open or close.
- Create or update location director accounts with the correct school email.
- Use password reset or setup workflows instead of sharing passwords in plain text whenever possible.
- Confirm each location user has access only to the intended center.
- Use audit logs when reviewing sensitive admin actions.

## Inquiry Forms

- Use the inquiry/embed tools to copy form code for a brand or location.
- Confirm each embed uses the correct location ID.
- After adding an embed, first verify the public script, location list, routing audit, and approved-origin `OPTIONS` preflight without creating a lead.
- Submit a synthetic inquiry only when the named test location and email/Google Sheet side effects are separately approved; then confirm it appears only in the correct CRM center.
- Confirm notification emails and Google Sheet backup are working.

## FTE Reporting

- Review weekly submissions from all schools.
- Watch missing-report alerts and escalation states.
- Correct inaccurate submissions when needed.
- Export CSV for offline review or backup.
- Use bulk import only when correcting or backfilling known historical data.

## ProCare Import

- Import one location at a time unless the bulk file is explicitly prepared for multiple centers.
- Require dry-run/preview review before committing data.
- Validate classrooms, families, children, guardians, staff, balances, and attendance after import.
- Keep backup files and batch IDs for every import.

## Payments

- Kid City USA pilot access is not billed until approved.
- Do not enable parent tuition checkout for a school until Stripe connected account onboarding, fee disclosures, refund policy, and support procedures are approved.
- Confirm Stripe status per school before allowing parent payments.

## Reporting

- Use CRM reports for lead flow, conversion, and school follow-up.
- Use FTE reports for weekly attendance/business snapshots.
- Use readiness/reporting views to identify incomplete school setup.
- Export reports before making major corrections.
